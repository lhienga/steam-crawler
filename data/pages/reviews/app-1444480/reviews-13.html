\r\n\t\t\t\t\tThis game is awesome!

When I was younger I dabbled a bunch in designing and implementing toy scripting languages and VMs to run them on. At least once I even designed a simple generic ISA that I intended to make a simulator for (I never actually finished that project). So as you may imagine, this game is pretty much my interests exactly.

I like how it introduces you to every component by lightly guiding you to make it from scratch using the components you have made up until then. I mostly breezed through this part since I was pretty familiar with the basic concepts, but enough information is given and the steps are small enough that a motivated person who is running into this kind of material for the first time should be able to progress by simply treating it as a puzzle game. (This is a brilliant way to teach people basic boolean logic by the way.)

For me the game really came alive when it got to the part where you actually built a computer. This is a lot harder than writing a simple VM, and a lot more fun :P Still, when I got to the section where you start over and build a more complicated and complete computer (the LEG, which is a great pun BTW) I discarded the framework the game encouraged me to make and designed a fully custom accumulator architecture very loosely based on half remembered concepts from the 6502.

A few notes:
  1. The "assembler" the program block provides is really crude, and not very useful for anything advanced. Expect to write your own external pre-processor or even a full assembler if your architecture is sufficiently custom and/or realistic. I don't really think this is solvable, there are too many possible ways to encode assembly mnemonics into hardware instructions. If you aren't up for writing your own external assembler, keep your designs simple.
  2. For some bizarre reason the "lab" sandbox level does not have tick acceleration like the normal levels do. After waiting around 40m for a simple software scroll on the 80x24 terminal to run, I started doing all my software experimentation in an actual level where I could boost the clock speed.
  3. I really wish the terminal block simulated an old hardware terminal. Old TTYs used to have their own screen buffers, and special hardware support for "basic" tasks such as scrolling, clearing regions of the screen, etc. Due to the way RAM works in this game you can't really do "cascading" memory banks like you would need to get proper smooth hardware scrolling and line blanking without it taking too many ticks and more gates than the computer itself. I'm trying to design a processor here, not peripherals.
  4. The poor terminal support extends to the keyboard input component. In the default mode you seem to get raw keyboard scan codes, but if you switch it to character mode you get some things in ASCII and others in whatever random format it decides to give you. For example, in character mode I would expect enter to give you character #10 (newline) or even #13 (carriage return), but instead it does its own thing. Even tab isn't a tab character. Character mode is useless, just use scan codes and build some kind of LUT into your input hardware to handle translation yourself until the game fixes it.

If you have any sort of interest in how computers work or even programming, I highly recommend this. The step-by-step breakdown is really well done and will take you from the point of no knowledge to a pretty good level of understanding. I have not got to the end yet, but so far I haven't got to a point where I felt each additional step was too big a jump that I felt I needed to go look at outside resources to understand what I needed to do which to me is the biggest triumph of this game. It tackles a difficult subject and makes it understandable. Not that the solutions are always easy, you have to think, it can be challenging, but the jump from each previous level isn't so great that you cannot figure it out if you understand the previous step.

Coming in with some programming knowledge is extremely helpful, as you should understand the basics quicker.\t\t\t\t\t
