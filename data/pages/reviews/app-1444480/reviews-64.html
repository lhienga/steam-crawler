\r\n\t\t\t\t\tCrashes frequently. Erases playtime. These are technical issues that can be fixed while a game is in early access. Minor, really.
The major problems for me are that while the concept is great, the game bugs out at working answers if they're not the ones it expects of you. Or skips basic knowledge on what Logic Gates are, why you're learning Truth Tables, how to apply those, and why in the machining of computers the NAND gate is used to simply build the same things with fewer Logic Gates. They just drop "You could use NAND Gates to do it all" skipping what that means, what the different gates are, or why you would do that. It acts as if it's being bottom up and step by step.. but it's not. It skips all kinds of essential steps and then faults you later for it if you don't guess which way the game rounded those edges. Like other players, I do see the potential here and the concept is something I've always wanted to see, so I'm hopeful.. but for the present offering, I can't recommend it.

\r\n\t\t\t\t\tTuring Complete is... Turing Complete! Unlike most tautologies, this one actually means something beyond being a trivial statement equivalent to a=a. The game directs you to make, from scratch, starting with nothing but a NAND gate, an entire, functional 8-bit computer, which theoretically can compute anything that can be computed. In other words, the advertisement for the game is 100% accurate. You really do build every part of the machine, from every logic gate, to the ALU, to registers, byte storage, etc.

There are only a couple exceptions to this rule, and in those cases you build something smaller first and then "an intern" builds the larger thing. An example is a puzzle where you build a 4-byte addressable memory system. After you complete that puzzle, "an intern" expands your 4 byte system into a 256-byte RAM chip. Incidentally, the memory puzzle is one of the hardest puzzles in the game, because it's the only one (at least so far in my playthrough) where they give you a limited amount of space to work, and you'll need all of it.

After you build a working computer, you're not done! There are still many more puzzles to solve, and you even develop your own assembly language to solve them. After the first round of the post-working computer puzzles, you expand the architecture of your machine to make it more useful.

I'm a programmer, so I didn't have a hard time understanding what was going on and what was being asked of me for each task. However, I also teach people to code sometimes in my spare time, so I think I have a fairly good grasp of the necessary skills and ability needed to tackle this game. For the most part, the game leads you very logically and methodically closer and closer to the goal of a working computer, in such a way that only the most basic skills and understanding are necessary to be able to continue progressing in the game and have fun. There is only one puzzle that left me scratching my head, but in a way I'm glad that they forced me to think outside the box (pun intended).

In my opinion, you will be able to play this game and understand it with only a few basic prerequisites. The number one prerequisite is binary arithmetic. If you don't understand binary logic/arithmetic, I think you're going to struggle with this game. I do think that it's possible to use this game to LEARN binary arithmetic, because they give you great tools to help you to do that, but realistically, I do think it will be a much harder challenge without that prior knowledge. The second prerequisite is simple logic, but if you understand binary logic, you understand simple logic, so it's kind of encapsulated in the first prerequisite.

So if the game sounds fun, but you're worried it might be too hard, I would say go for it anyway. Just be prepared for a steeper learning curve if you are not familiar with binary logic. Even so, this game is the funnest way to learn binary logic that I can think of, so if that's something you've been wanting to learn, TC is a great tool for you.

The UI takes some getting used to, but once you do, it's actually pretty great. It's hard to imagine a better UI for doing this kinda stuff now that I've gotten my bearings. However, I did end up assigning some macros to my mouse (I have a logitech g502 which allows me to program macros for specific games). This is my recommended way to play the game. I assigned some buttons to commands that make it easier to place and move wires around.

On top of all that, I also dabble in electronics, and the components in this game are similar enough to real-life electronics components that I now feel confident I could build a REAL computer, from scratch, which truly blows my mind.

My review wouldn't be (Turing?) complete without listing some negatives. So here they are.
- A couple of the puzzles have vague wording that takes a while to understand. Mostly they are pretty straightforward but I did have trouble getting the point a couple of times.
- There are very few players for this game, apparently, so there's not a lot in the way of internet resources if you get stuck
- A few of the design choices are questionable/slightly annoying. I haven't had any huge annoyances, but there comes a point in the game where they fix some components in place and you're not allowed to move them. They do this for a good reason, though, which you learn later.

In summary, if it sounds like fun, buy it immediately, and if you're on the fence, jump over the fence immediately. You won't be disappointed.\t\t\t\t\t

\r\n\t\t\t\t\tVery very cool game.

The difficulty curve seems quite balanced so far, alternating between building components with logic gates, building CPU with component and writing programs with assembly makes the "gameplay" diversified.

The game makes you realize that you probably already understand all the building blocks to build a computer, it just empowers you with tools to do it fast, removing any manufacturing constraints.

There are still a few tiny flaws due to the beta state of the games: a few random crashs while moving objects (which are not so annoying since it starts in about 2 seconds and seems to save after each action) and sometime lacks a bit of documentation about what to do or some specifications that are just given in a former level intro dialogue (eg. ALU specification).\t\t\t\t\t

\r\n\t\t\t\t\t"Turing Complete" is a game, not a full-blown simulator (like "Digital" by hneemann) which you could use for the same end, and you won't have a computer in hardware to show off like you would if you were actually building the thing with the 7400-series (and such).

Being a Game TC gives you /reasons/ to build stuff like a decoder, an ALU, a negator and so forth. It looks good. It gives you a nice overview how far you have come, the alien makes funny comments (at least some of them are...), It "teaches" quickly and gives you satisfaction - at least if you understand what it wants...

So let's come to the downside: It is not a very good tutor for certain problems. Notably the counter level has driven me nuts before I realized, that everything I had done was absolutely okay, only my testing had been erroneous. Besides I had thought about building a counter based on ms-d-flipflops or suchlike before I got the tip to work with a register feeding into itself by browsing the steam forums.

I am only 7 hours into the game. So maybe some levels leave you without any hint or there may be other problems to come. But altogether it is a very enjoyable game and - to me - absolutely worth the money.

I still have to get my HiDPI-problems with the game fixed, the gui (components, menu) is too small, but that is my personal problem, I guess...

TLDR: Building and programming your own computer and knowing where all the bits come from (and go to) is fun!\t\t\t\t\t
