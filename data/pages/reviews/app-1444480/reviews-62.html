\r\n\t\t\t\t\tThis game is amazing if you've ever had any interest in building a computer from scratch, like a Minecraft redstone project or a breadboard computer. I recommend grabbing a copy of The Elements Of Computing Systems ("Building a Modern Computer from First Principles", also known as "NAND to Tetris") to keep as a reference if you get stuck.

I love the control that the game gives to you in later levels. Choosing my own opcodes and designing instructions to solve the assembly challenges in a more efficient way felt great.

Primary complaints:
- At the end when I had a functioning computer with RAM, stack, function calls, etc, there were parts of the machine that I didn't feel like I really "built" -- in particular, the jump from 4 byte RAM to 256 byte RAM is maybe too sudden. If there's any way to add additional levels (even if they're optional) to help the player see a pattern that can be scaled, I think that would help. Basically, I solved the 4 byte RAM problem, and was then given 256 bytes of RAM, but my solution to the 4 byte RAM was so awful I am not convinced I could have scaled it to 256 bytes.
- I don't think the game should hide components that you've unlocked in later levels. Even levels along the same "tier" should let you use components that were unlocked on that tier. The whole point of the game is building upon abstractions, so don't take that away from the player. There were several levels in the end where I had either solved a prior problem and unlocked a component that would be useful, or had outright created a custom component in the component factory, but was prevented from using either the previously unlocked component or my custom component in the current level.

Minor complaints:
- The game has crashed a few times for me. This is probably just a symptom of EA.
- The custom labeling of instruction bits seems bugged / undocumented. All of my labels only fit on the last bit.
- The "manual" is used to document many UI features, but just as often those UI features are documented in tips on level prompts and seemingly nowhere else, which forced me on more than one occasion to go back to an earlier level just to reread the earlier prompt to remember a UI feature.
- The assembly editor doesn't seem to have support for the conditional bits introduced for immediate values, so it's awkward to define N variants of an opcode to cover add vs add w/ constant, etc.
- Ditto for opcodes that don't use all of their arguments. I often found myself writing things like "jmp _ _ label" for unconditional jumps.\t\t\t\t\t

\r\n\t\t\t\t\tGood game overall, still definitely EA but definitely in a completable state.

Cons:
- Regular crashes to desktop when moving things around the editor - occurred about 6 times playing through to the stage of having a working computer. Never lost any progress as a result though. Also 1 crash to desktop while running the computer itself.
- Layout tools aren't fully refined yet. It's easy to accidentally join a bunch of wires or have troubles routing them the way you want. Similarly moving things around is a bit hit and miss and can get very iffy in bulk.
- Instructions aren't always the clearest and are a pain to try to reference back to as you're building the design, especially in later levels where it takes up multiple pages of popups.
- The game starts forcing layout decisions for initial versions of components towards mid game resulting in a lot of "I just finished the part now let me go back and redo it since I can actually lay it out right now" work.
- The circuits path isn't refined yet, e.g. there is no bitwise AND/NAND level and you need them for the ALU... which once you complete you can make custom blocks like the bitwise AND/NAND you can redesign the just finished ALU with. Similarly there is no subtract element even though it has the lead up for how to do it.

Pros:
- Very approachable means of going from absolute basic circuitry to programming in a single tool
- Gamified mechanics to make it all more interesting
- Decent price
- Pretty well thought out and well implemented, even if not fully polished.\t\t\t\t\t
