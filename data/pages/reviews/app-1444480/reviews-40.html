\r\n\t\t\t\t\tthis is a thumbs up review because there is no "it depends" option.
I have a learning disability that gives me profound difficulty with math - with friends who are computer scientists riding along who could explain concepts, I can make progress. Using this mainly to give feedback, because I love what this can do!

- The controls are very opaque and would benefit from an optional tutorial to learn how to use them. Shift+Click to select is clear, that I need to then release these keys and click again to drag things is not, for instance, and until I realized this it was not clear what was going wrong.
- please just permit the organisational tools to be used from the beginning. Access to colour coding wires straight away would have saved me about an hour of confusion and frustration, having to put my finger on the screen and trace where the lines were going.
- if possible, please consider defining terms. At no point was it explained what a "bit" is, before I was told that now I'd be using a "byte", nor how many "bits" went into a "byte". This is super easy to overlook, but it is important.
- Related to the above, if an input is arbitrary and could be anything, folks like myself would benefit strongly from being told that. I spent a long time staring at the intro to byte splitting and bit combining completely confused about why the input was 123, and wondering if there was some core concept I hadn't picked up on that explained why it was 123.
With additional, external aid, I can use this to learn things that I never thought I'd be able to grasp. In terms of it as a stand-alone teaching and puzzle game, it's currently not useable for me personally for that purpose. I need to know how and why something works to be able to draw the rest of the metaphorical owl, without that I will have no clue how to proceed and not actually learn anything.
Edited to add: the narrative is actively hostile, as a warning. The framing device is already very dubious for a teaching tool, but having the awful muppet man neg you constantly about whether you're even sapient or not or comparing you to a dog is like. Very not great while trying to learn complicated subjects. Genuinely would advise removing or strongly adjusting these elements, they don't add anything.\t\t\t\t\t

\r\n\t\t\t\t\tI've done a thing. I've built a computer.

It's not that hard especially if you have some background in CS.

You start with the NAND gate. The first task is to build other gates with NANDs only. Then you create memory and basic arithmetic operations. Finally, you assemble the components into a working computer.
You develop your own syntax (language). You program your computer using your language. There are even programming challenges for you to solve using your own programming language.

The process is amusing. It doesn't feel like studying but more like playing and learning. The game has a plot and the jokes are funny.

\r\n\t\t\t\t\tIf you are interested in learning about the fundamental building blocks of a computer in an interactive way, you might want to give this game a try, but overall I can't really recommend it.

I have a degree in computing, the concepts therefore aren't new to me. If you have no previous knowledge, you will probably find yourself scratching your head quite a few times, but this is just due to the nature of the topic. There is a manual, which provides some additional info, some levels have additional hints and some even link to a video explanation and solution on Youtube, but overall I found the help a bit limited. I don't count this as a negative, though. Coming up with your own solution is the goal after all. However, sometimes the goal itself isn't really defined clearly enough.

The biggest gripe I have with this game, is that the interface feels very clunky. While it's okay on the earlier levels, it get's really frustrating to reorganize components in the later stages. Rechecking, what you are supposed to do is annoying, since you need to reopen the level log, which opens a small popup with multiple pages. It would be much easier to have the description open all the time somewhere. In the first assembly levels, you need to define every instruction manually, by copying the instruction codes from one window to another. These and others are many small things, which reduce my enjoyment of the game greatly.

Oh, and there's also some bit of story, but it's really bad.
