\r\n\t\t\t\t\tThis is probably the best game I have found that demonstrates EXACTLY how a computer can be built up completely from simple logic gates. The interface is very clean and the UI is intuitive. The "story" is good enough to give a good chuckle every now and again, though the game understandably focuses more on the puzzles than telling a narrative. I think the only critique I could give is that this is not a game for those who don't enjoy difficult puzzles or get frustrated when trying to work through complex logic problems. While the game will hold your hand at having you build each little piece of the computer in succession, the puzzles can get quite difficult to think through at times. However there is a strong sense of enjoyment in finally completing an extremely difficult level.

One thing that surprised me about this game, was the amount of flexibility given in how a puzzle is solved. For example, there are a few levels in the middle where you write some assembly code to be run on the basic computer that you have designed. At this stage, the game has not yet walked through a few more specialized components that are not required to finish the level, but I did find myself wishing that my computer had one of them in particular so that I could perform a particular operation in my program. Wouldn't you know it, the component designer allowed me to design the part, plug it into my computer design, implement an opcode for it in my program (borrowing one of the "unused" opcode slots) and then use that opcode in the assembly I was writing. Despite the fact that I was using a mechanic that should be unexpected at that stage of the game, it was allowed to happen and was counted as a valid solution.

Overall an extremely enjoyable (and educational for that matter) game that I would HIGHLY recommend to anyone who has ever wondered how simple binary logic operations can be used to make a computer run.\t\t\t\t\t

This game is captivating!

Complexity:
While the concept of logic gates was familiar to me, I started struggling after reaching the second chapter. Especially the memory section seems to be hard for a lot of people and I think most of the games shortcomings are in one specific puzzle of that chapter.
In said level you need to make an S-R-Latch save one bit of information. While this seems fairly straightforward, it is way too difficult because the workflow necessary to create the right circuit is really only used in that level. Or for short: the complexity spikes in a short amount of time.
To be fair, the dev put a reminder in the level to get familiar with the S-R-latch first, but I would like to see maybe one more level involving the S-R-Latch to force players to get familiar with it.

And that is exactly what has been done so well for the rest of the game. New concepts are always shown first, then tested and then applied. The game is incredibly good at giving you the right kind of difficulty, each and every level. There is literally only that one level I would like to see tweaked.

Bugs:
There are some crashes here and there, but while usually that would be a gamebreaking thing for other titles, it really is more of a slight inconvenience as every progress is autosaved and the game starts up incredibly quickly.
I had one graphical bug, but restarting my pc fixed it (and that really only happened once).

Flowstate:
This game isn't fun, it's something even better: it's absolutely captivating. I've rarely fallen so easily into a flowstate before. I can't really communicate how exceptional this game is at making sure you are always familiar enough with what you are doing. There are levels where you probably gonna need an hour to complete them, at least if you have never seen assembly (or are just playing around while solving like me), but you are never stuck in a sense that you have no direction. I was almost always in the process of doing something intentionally instead of just trying out mindlessly (except for some logic gates, but to no avail).
If you have ADHD, you can expect a lot of hyperfocus.

Summary:
I can really recommend this game if you are in compsci (or anything that has to do with math/computers), not only because it may be educational, but because you'll most definitely have a good time playing it. This game is still in early access, but you're not gonna notice that except for the occasional crash. I love the quirky story, even though it's not really important to the gameplay. It's definitely better than just some plain text and it actually adds some immersion to the already incredible gameplay loop.
It is worth every cent I payed for it and I will probably gift this game to some of my friends if they want it. 9/10\t\t\t\t\t

\r\n\t\t\t\t\tvery cool and interesting game for anybody interested in computers, circuitry, logic puzzles or redstone

i will say this game has a *very* steep learning curve, and by curve, i mean cliff
unless you know how to read truth tables, you will struggle
i feel like the game could really benefit from showing the desired inputs and outputs in action (not on a table, but just how they should respond) or by laying out some wire and leaving the question to be which components to attach (only in the earlier levels though) or something to make it easier for newcomers

i really do like how the game teaches teaches the components by first making you build them, then letting you use them as a simple block
it really is amazing how much you can make with a bin full of NAND gates

also this game would really benefit from an easier way to select components, maybe the 1-9 keys and scrollwheel to switch between gates, memory, etc... categories
i find it kind of flow breaking when i need to move my cursor away from what im working on to select a different component

also also a sandbox type mode, with all the stuff you've unlocked so far and configurable inputs/outputs (there may already be one, im only halfway through arithmetic/memory)\t\t\t\t\t
