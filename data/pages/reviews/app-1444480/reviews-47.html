\r\n\t\t\t\t\tTuring complete is most complete logic gate simulator I've seen for a long time.
It extends the learning of computers beyond the logic gates to the macro functions of a processor, up to assembly code programming.
Through a series of levels more and more complex, it introduces the basic blocks of circuitry that you must reuse in levels beyond.
For instance, one level asks you to add two bits with logic gates, then next level ask you to build a half adder then the 1 byte adder.
A this time, it provides you with basic functions as component that you built previously.
Progressing in this game empowers you with knownledge of electronics.
I can't recommend enough this game to people who wants to understand how our little devices are running at the lowest level.

\r\n\t\t\t\t\tThis game is great for if you want to learn the basics of how CPUs actually work under the hood. It's obviously not as in-depth as what you might find in an undergrad course in the topic, but could absolutely be used to give you a head-start in the right direction.

I'd recommend supplementing this game with Silicon Zeroes. Finish this first, then play SZ if you enjoyed it. TC will take you from NAND gates, through components, up to a full CPU. Meanwhile SZ will start out with basic components already implemented, but will take you all the way up to vector processing and basic pipelined execution, which Turing Complete can't simulate.

The simulator is also pretty good, but not quite as powerful as Virtual Circuit Board's. That being said, unless you already have some experience you'll need to play through TC to make sense of VCB in the first place.

Overall, get this game if you have even a passing interest in computers and how they work. The lessons you learn in this game will be *incredibly* valuable unless you're familiar with the topic already, and optimising your solutions will be incredibly satisfying regardless of whether or not you know what you're doing.\t\t\t\t\t

\r\n\t\t\t\t\t** IMPORTANT:
~ If you get frustrated, don't give up,
On the Turing Complete 'Play' page, Click the Turing Complete 'Guides' -> 'Secrets' -> '100% Guide' section by 'StormSurge'. It is the most complete & clear solutions list I have found.

~ Don't rely on the original hints videos in the program, they are such poor quality that they can't be easily read. And most other videos like the original hint videos don't offer much of a clear explanation, also circuit images are not showing up from other Steam player sources.


** UPDATE:
~ PROS: Teaches basic computer programming skills, Sandbox mode, Schematic Hub.
~ CONS:
1) This is not a program for those who don't have some background in digital logic.
Because the original hints/solutions videos are narratively implied & not clearly expressed.
Even the alternate solutions online may not be clear.
So some increasing degree of difficulty & getting stuck will incur, as the levels get higher.
2) The Play Campaign is more of an intimidation test than an educational game, by limiting available logic resources, unlabeled pins, aggressive intro comments by the 'alien' character, & by limiting working space.
It is not fun to play, unless you are really motivated to overcome the challenge.

The fun is in the sandbox, & custom saved components.

** CONCLUSIONS:
~ This 'game' is intimidating for the inexperienced, & a challenge to the practically experienced.
~ Someone who is used to drawing circuit schematics & using ic chips with their labeled designated pins, can benefit from using simplifed programmed symbol icons instead.
~ This 'game' has been out for a year now, but is still listed as 'Early Access' on Steam.
So this clearly shows that there are still some bugs that need to be addressed.
~ The 'Play Campaign' in this program is more like an awkward challenging programming game than an educational curriculum, because the GUI configuration has been arbitrarily diffused for that purpose.
~ But overall, it comes out & about as an accomplishment for the determined & perseverant.

** UPDATE:
~ So I finished 'Turing Complete' in CPU Architecture, then finished Add 5 in Programming, then went back to Turing Complete to see it go thru again, but it didn't.
Therefore changing the addresses in the Program table in the 'Program' level, doesn't allow one to go back to double check work in Turing Complete via the first CPU Architecture level.

** UPDATE: I got as far as Calibrating Laser Cannons.
So when it came to the Assembly Code Editor, it didn't show the positions or what instruction code I was inputting, so that was a hassle, therefore I had to go back & memorize what the instruction, registers, & input/output codes were, to get thru.

** UPDATE:
~ I haven't played the game lately as of yet, because I am busy playing some other games. But I am also studying & going thru all the programming instructions available, as well as maybe coming up with some of my own.

\r\n\t\t\t\t\tThis game has been an interesting experience. I found it mostly pretty easy, but I also previously studied assembly programming and circuit logic in college. I did find a few of the levels a bit challenging, especially when the space you are given to work with is limited. I suspect this game would be considerably more difficult for people who don't understand logical algebra. Most of my fun, though, was derived from actually experiencing wiring together a functional computer piece by piece from logic circuits, which I could then program and have do whatever I want.

I only observed one minor bug in the game; it seems to sometimes get confused when dealing with components that deal with both 1-bit and 8-bit wires. It would suddenly change a whole wire into an 8-bit wire when a 1-bit wire would have been fine and none of the inputs or outputs were 8-bit. Not that that really changed anything, as the signals still functioned identically, but from an aesthetic point of view it was mildly frustrating on a level or two.

I also have some difficulty understanding how the RAM module works. With four inputs and four outputs, I thought that I could be clever and create RAM that handles 4 bytes of information at a time instead of just one, but it seems to utterly ignore inputs 2-4, regardless of what I set the data width to in the module settings. I was able to complete the RAM level by just using one byte at a time, but I'm still a little annoyed my 4-byte solution wouldn't work.

The UX was mostly pleasant and worked as expected once I had gotten the feel for a few things. I do feel like the marquis selection doesn't work completely intuitively; I wanted a way to drag a new selection from scratch without clearing the existing one, but too many times I shift-dragged over a selection only to accidentally move components I had not intended to because I forgot that I had selected them before. I also found that the wire color box was easily unintentionally triggered, and it took some experimenting to figure out how to get it to go away again without picking a color. I do wish it was possible to recolor wires that already exist, rather than having to delete them and redraw them.

Overall, a fun and educational game. Get it and try it out, if this kind of thing interests you.\t\t\t\t\t
