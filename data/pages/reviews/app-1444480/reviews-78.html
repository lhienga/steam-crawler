\r\n\t\t\t\t\tI agree with the other comments that this is not as polished as the Zachtronics offerings. To a certain extent it scratches the same itch, but it has major shortcomings.

I'm writing this as both an educator and having education in electronics. This smells like educational software, though it doesn't teach you the simple tools that you learn in college to solve problems like these (especially Karnaugh). It has now made an attempt at De Morgan - though perhaps not the clearest one.

It has thankfully fixed(ish) the wire contention problems. "Short circuits" are disallowed. However, the problem description is at the wrong level of abstraction - since there is no description of a "circuit" at all, the user is left wondering what specifically a "short circuit" is. "Output conflict" is perhaps a better-suited description.

Other assumptions would lead to poor understanding of real logic hardware and bad habits. Leaving a floating gate is (almost) never the way to generate a zero. If you want to provide a magical "0" part without explaining what ground is, that's fine; but don't offer an explanation for a logic zero that's misleading.

The UI is slightly clunky. Drawing a byte bus has an animation of data flow (good) that seems to have a non-deterministic direction (not good). The user can't do simple left-click selection of wires or buses, and only nodes are selectable/movable. Some of this is excusable because the game is an early release.

One major annoyance is that the simulation halts as soon as the first failure is encountered. This prevents seeing the whole behaviour of the circuit and fine-tuning it for specific edge cases. At least you can step through to the end.

Thankfully the assembly editor's operator precedence issues have been fixed. However, especially for an application targeting (presumably) non-technical users, the error descriptions are quite lacking - "something went wrong".

Quality-of-life features are either awkward and unexplained - like drag-to-pan in the menu requiring middle-click - or missing. When a group selection is made, dragging on a connection node should move the group, not make a new wire. Dragging in empty space should do what 'shift' does now (selection), not make a new wire. There's a bug where the wire labelling tool seems to randomly fail to turn off.

It's clear that the developer is actively working on issues and this game is making progress. I still think it leads non-experienced users down a lot of blind holes and incorrect or unstated assumptions, but overall it's fun enough that it's crossed the line from don't-play to play-as-long-as-you-already-understand-electrical-engineering.\t\t\t\t\t
