\r\n\t\t\t\t\tTo preface all of this, I have a gaming laptop. It's by no means a bad computer, but it's a couple of years old, and it is still a laptop. Six cores, 16 gigs of RAM, and a laptop 3060.

That said, as I write this, I'm watching a four million unit battle unfold. On high settings. It isn't running very well, as one may imagine given my hardware constraints, but it is running, and it has been running stably as hordes of giant bugs overrun two strongholds and bear down on a third, eliminating an entire infantry division who countercharged as they went.

That is to say, this game performs really, REALLY well. Like, absurdly well.

This is an excellent, well designed product. It promises epic scale battles, and it delivers. Any battle you can imagine (so long as you can find a mod for it). With that said, however, there are SOME changes that need to be made, and some that I personally would like to see.

Needs to change
- The way that remaining troops and kills are tracked is really confusing. A better UI for that, probably with troops remaining and current kill counts separated out, is needed.
- If you change a keybind in settings, the UI doesn't seem to reflect it. This needs to be fixed, if only for clarity's sake.
- You should be able to change the audio settings from inside of the round. C'mon.
- Better support for modding. It's got some ok stuff now, but having to install a Unity editor, and then troubleshoot the very old version of the editor you have to use when things don't work right, is a hassle.
Would be nice
- If units had some concept of what cover was.
- If units could use the small buildings like bunkers for their intended purpose, or otherwise garrison / occupy buildings on the map.
- If vehicles behaved like vehicles rather than units, and could be mounted, dismounted, repaired, etc.
- Health regen.

Other than these things, this is a really excellent product! I'm very excited to see where it goes.\t\t\t\t\t
