It has a much more consistent Medieval theming than either of them, which is something I like. I found building walls around the settlement to be much more critical in this game. Took about 50 hours and 7 towns to feel like I fully had a handle on the game mechanics while still in EA. Like Rimworld & Embark there's a good amount of replayability based on on the combo of starting conditions. I'd recommend this to anyone who wanted something like those two games with a stronger Medieval theme.\t\t\t\t\t

Edit:
Update 7 fixed the problem somewhat - but it's still a little frustrating, I do believe the devs will eventually solve this but I cant recommend the game really in it's current state.

Let me start by saying this is a good game and I really like where it's going, but it has some glaring issues which will frustrate the hell out of you.

the villager job prioritisation is very frustrating.

Basically you can give each villager a set of priorities that they will use to determine what to do next, this makes sense as you'd want the villagers to do more important things like make food rather than cut down trees etc.

The problem is the way this is set up makes for some very frustrating behaviour with how the villagers will prioritise jobs, because this system is really simple - villagers wont take into account distance or the time they have left in a day or anything like that when choosing a job. So you get situations where one villager will take 8 hours to run all the way across the map to harvest some wild hay, only for them to leave the hay on the ground and then run all the way back to home, then days later some other villager will make the same trip to pick up that one piece of hay.

- you'll have hunters that will run off to shoot a deer only to leave the carcass to rot/be eaten by wolves while villagers at home are moving sticks from one pile to another,
- you get villagers that will run for two hours to move a stick off of a farm plot, then run all the way back to base to milk a goat, then run all the way back to the same plot to move another stick.
- I had a crop blight happen, so I set "cut plants" to priority 1 and marked all the blighted crops, the villagers removed them and then promptly ran off to my tree farm to to cut trees all day while the blight continued to spread.

sure you can prioritise "Haul" so that moving stuff is top priority, but then villagers will run off to no where to move a rabbit carcass/dirt/bone to your base and you'll starve to death cause everyone's too busy to farm, closest I've got to solving this is by having some dedicated haulers but it still means multiple people are doing a job that one could.

I'm not sure if the solution is a more granular prioritisation system (it's already pretty tedious) or perhaps the devs need to take inspiration from the corporate world and graph each job in a matrix with it's value vs difficulty and have the villagers prioritise jobs that way. desired state being a villager runs to the middle of nowhere to hunt a deer, kills the deer, sees that haul prioritisation is low but has to run back to base anyway so the job is "easy" and does it anyway despite prioritisation.

anyway if they ever fix this the game would be an 8/10 for me, it's done everything else right but this is a core mechanic and I just couldn't watch the villagers be stupid anymore.\t\t\t\t\t
